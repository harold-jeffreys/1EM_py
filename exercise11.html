Exercise 11: Prime numbers

This exercise is intended to give you more programming practice, and is in two parts. The first part is to make sure that you are able to make use of functions. The second part is optional, and is to do something neat with the subprogram that you write in the first part.

Part 1

The objective here is to write a function that takes an array of integers, and returns the index of the first non-zero element.
Your function only needs to work for a rank-1 array (i.e. with one index, or a vector-type array). On the other hand, it should work with a rank-1 array of any length.
Once you have written your function, write a program to go with it that calls it with some different arrays, whose first non-zero element is at different positions and check that it works correctly.

Part 2

Prime Numbers

If you get here well before week 9, try this second part of the exercise. The goal here, is to write a program to implement the Sieve of Eratosthenes, a method to determine the prime numbers. I will describe briefly how the method works, though referring to the Wikipedia entry is also helpful.
The algorithm works by starting from a list of the first n integers. This will then lead to all primes ≤n. The number 1 is immediately discarded from the list, as being a special case. The next number is 2. Since it is still in the list, it must be prime. All multiples of 2 are then crossed off the list. Since they cannot be prime.
The next number, after 2, that is still on the list is 3. That must be prime, but all multiples of 3 still in the list are crossed off. The next number still on the list is 5 (because 4 has already been crossed off). 5 must therefore be prime, and all multiples of 5 are now crossed off, if they have not already been. This continues until you have worked through the whole list (actually you can stop after n/2 and still get the right answer).
Write a program which has a subroutine in it that accepts a 1-dimensional array of integers, in order (i.e. a(1)=1, a(5)=5 etc), implements the sieve of Eratosthenes in such a way that it sets non-primes to zero before sending it back to the program. Depending on how you choose to write your algorithm, you might find your function from part 1 to be useful.
