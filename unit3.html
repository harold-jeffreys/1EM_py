Unit 3 - Conditional statements.

Introduction

Until now, all the programs we have written were sequential and linear : the sequence of instruction executed was always the same, regardless of the data. In programming, it is very often required to execute different instruction depending on the data we have. Let's take a very simple example, imagine we want to convert length units between millimetres and inches. We can imagine how the program might be structured:

  1. Ask the user if they have a value in units of inches or millimetres.
  2. Ask the user a value.
  3. If the conversion is from inches, display the value converted in millimetres.
  4. Else, display the value converted in inches.

With what we learned before, we should be able to do everything except the if/else part. This requires what computer scientists call conditional statements. For that, we will use the if, else and elif statements in Python. You can learn more on the official documentation.

The general form of a conditional statement is the following in Python : 

if condition:
  do_something
else:
  do_something_else

Note the indentation here! Some programming languages use brackets, or key words to group statements. Python uses indentation.

Boolean values

Let us analyse the components of the if statement. The condition has a very specific type that we have not yet covered about until now : it is called a boolean. A boolean can only take two values: True or False. When writing an expression that resolves (i.e. results in) to a boolean type, Python evaluates the truth of the statement and returns either True or False. Let's see an example of condition : 

a = 10
print(a > 0)   # True
print(a < 0)   # False
print(a == 10) # True
print(a != 3)  # True

There are a few testing operators that allow you to create conditions with any kind of variables. Here are the most common ones :

Operator Meaning Comment
a == b Equality testing Be careful not to mistake this with a = b which is the assignment operator. a = b will give a the same value as b while a == b will return True if a and b are equal.
a != b Not equal to
a < b Less than
a <= b Less or equal
a > b Greater than
a >= b Greater or equal
a in b Belongs to If b is a collection (list, tuple, string, dictionary), then a in b will return True if the value of a can be found in the collection e.g. 1 in [2,3] returns False, but 3 in [2,3] returns True.

It is possible to combine boolean expressions to form more complex tests. Consider we have two boolean values a and b, we can combine them to form another boolean value with the following operators :

Operator Meaning
not a Inverts the value of a. If a is True it becomes False and vice versa.
a or b Returns True if either or both a or b are True
a and b Returns True if both a and b are True

With these operators we are ready to tackle every single condition testing.

Blocks of code :

As you can see on the example (and might remember from unit 1) : after stating the condition, we have shifted the code to the right by adding a few spaces before. This is what we call indentation. In lots of programming language, indentation is only a cosmetic feature, helping you to read the code. But in Python, it has also a syntactic meaning : indenting creates a block of code. Python then knows what you want to be executed in your conditional statement, and what's to be executed after the conditional statement. Let's take two very simple examples to show you what blocks are :

Example 1

a = int(input('Enter an integer value : '))
b = int(input('Enter another integer value : '))

if a < b:
  print(a, ' is indeed less than ', b)
  print('Basic addition : ', a, ' + ', b, ' = ', a+b)

Example 2

a = int(input('Enter an integer value : '))
b = int(input('Enter another integer value : '))

if a < b:
  print(a, ' is indeed less than ', b)
print('Basic addition : ', a, ' + ', b, ' = ', a+b)

In the first example, the two print statements belong to the block of code of the if-statement. So they will be executed only if a is less than b. In the second example, you can see that the print displaying the sum of a and b is at the original indentation level. That means that this print statement will be executed regardless of the truth value of a < b, after the condition.

Back to the example

Now that we know all of this, we can write our unit converter in Python : 

conversion = input('Converting from mm or from in [mm/in]? ')
value = float(input('Value to convert : '))

mm_to_inches = 0.039 # We create a variable to hold the conversion factor
    
if conversion == 'mm':
  print(value, 'mm ~', value * mm_to_inches, 'in')
else:
  print(value, 'in ~', value / mm_to_inches, 'mm')

The elif statement

As you can see, we execute different versions of our code depending on the value of conversion. Now this is not completely satisfying. If the user enters 'km' for instance, the code will still treat the unit as inches. We want our user to only enter 'mm' or 'in'. That means we need a third condition :

  • If conversion is 'mm' do something
  • If conversion is 'in' do something else
  • If conversion is none of the above, then print an error message

The elif statement allows us to do so by chaining condition tests : 

if condition_1:
  # Do something if condition_1 is True
elif condition_2:
  # Do something if condition_1 is False but condition_2 is True
elif condition_3:
  # etc.
else:
  # Do something if all of the above are False

Knowing this, our code can now be completed : 

conversion = input('Converting from mm or from in ? ')
value = float(input('Value to convert : '))

mm_to_inches = 0.039 # We create a variable to hold the conversion factor
    
if conversion == 'mm':
  print(value, 'mm ~', value * mm_to_inches, 'in')
elif conversion == 'in':
  print(value, 'in ~', value / mm_to_inches, 'mm')
else:
  print('Error, the conversion unit should be either mm or in')
