Unit 9: Algorithm Design: Planning solutions to problems. Flow Charts.

As you start to write more complicated programs, it is worth planning first on paper how you might want to structure the program to solve the problem at hand.
As an example, let's say we want to write a program to find the real roots of quadratic equations. You will probably know that the formula for solving quadratic equations of the form $$ax^2+bx+c$$ is $$x=\frac{-b\pm\sqrt{b^2-4ac}}{2a}.$$
The nature of the roots will depend on the discriminant, Δ=b2-4ac.
  • If Δ>0 there are two distinct real roots.
  • If Δ=0 there is a repeated real root.
  • If Δ<0 there are no real roots.
To plan the program, we will highlight two methods. One is using pseudocode and the other is with a flow chart.

Pseudocode

Pseudocode is the name given to a way of writing, in a very human-readable form, the way a program has to work to achieve the desired result. It has elements of looking like real computer code, and is a kind of stepping stone between the thought processes about how to solve a problem, and the final result. We might write pseudocode for the above problem as 
    
print instructions to user
read user input
calculate discriminant
if discriminant is negative 
  print message saying there are no real roots
  
elsif discriminant is zero 
  calculate repeated real root
  print message giving repeated real root
else
  calculate two separate real roots
print message giving the two roots
As you see, we've used some key words to highlight the structure of the desired solution, but this still needs to be actually turned in to a real program. So the next step is to take the elements of your pseudocode, and turn it into actual computer code.

Flow Charts

Flow charts are graphical ways of representing the design of any kind of process, including computer programs. They are like pseudocode, in that the purpose is to think in a quite general way about the things that have to occur in your program and in what order. A typical set of flowchart objects are
  • Ovals to indicate the beginning and end of the program
  • Rectangles to describe actions
  • Diamonds to indicate choices
  • Parallellograms to indicate data input and output
