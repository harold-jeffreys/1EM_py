Exercise 11: Consolidating

A rational type

Fortran does not handle fractions neatly in the same way that, say, an up-to-date hand-held calcualtor can. In this exercise, the idea is to create a user-defined TYPE which stores two integers, such that one is considered the numerator of a fraction, and another the denominator.
With that type created, you should write a program which lets the user do the following things, by offering a suitable menu:
  • Simplify a fraction
  • Add two fractions
  • Multiply two fractions
  • Quit the program
Note that to simplify the fraction, you will need to work out the greatest common divisor of the numerator and the denominator. The standard way of doing this is to use Euclid's algorithm. It is worth putting the TYPE declaration in a MODULE, as detailed in Unit 11, so any subroutines or functions can have the TYPE available to them more easily than defining it several times in different places.
Once you are happy with your program, you can try any of the eXtra exercises (X1, X2 etc) linked to in the menu bar to the left. They can be attempted in any order.
