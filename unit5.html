Unit 5: Functions and modules

Introduction

As we have seen with the previous unit, loops are a very useful way to factorize code. However sometimes loops might not be sufficient to encapsulate correctly pieces of code you want to reuse. Or maybe you want to use the same piece of code in two very different places in your program and loops cannot help. That's why we have functions and modules. Long story short, functions are reusable pieces of code, just as functions in mathematics, that will apply a treatment to input data and return output data. Modules, on the other hand, are collections of functions that you can group together in a single file, and that you can reuse in other python programs.

Functions

As said before, functions are reusable pieces of code that you can call. A function takes zero, one or more input parameters and can return a result. You can see functions in programing as their mathematical counterparts. As a matter of fact, a lot of mathematical functions are available in Python for you to use. We have already used functions in previous units, but without explicitly mentionning they were specific objects of Python. For instance, print or enumerate are functions.

Now that tells us how to use a function, but not how to create one. Let's see a very simple example of function that will take two parameters and adds them together :

def add(a, b):
    return a+b

print(add(10, 12))    # 22
    
my_var = add(1, -1)   
x = 10
print(add(x, my_var)) # 10
    
print(add(1, add(2, 3))) # 6

In this example, we use the keyword def to create a function. def is a reserved keyword that means you should never use it for something else than defining a function (and especially not as a variable name !). The definition of a function always has the same form :

def function_name(parameter1, parameter2, ...):

As for any block of code, the contents of the function is indented so that Python knows where the function begins and ends. In the function, we use the special keyword return. This keyword, as the name implies specifies the result of the function. Here, we want add to return the sum of a and b.

The following lines show a few ways of using the function. Note that since we are not considering any type in the function, we can add any object for which the addition has been defined. For instance, look at what result add('Hello ', 'World') is giving you ! Functions are a nice way to abstract operations and add semantics to your code : with functions you group pieces of code with a single intent, explaining clearly what you mean to do. This is important when developing code that is going to be shared to users or other developers

Parameters in a function can be provided with a default value. These parameters can be omitted if the user does not want to provide a value different than the default value. If more than a parameter has a default value, the user can specify which parameter is going to take a specific value when calling a function : 

def add(a, b, c=0, d=0, e=0):
    return a+b+c+d+e

print(add(1, 2))
print(add(1, 2, 3, 4))
print(add(3, 2, e=6))

Important note when using default values for parameters, always make sure that all parameters without default are first in the function or Python will return a syntax error. For instance def my_function(a, b=10): is valid while def my_function(a=10, b): is not !

Functions are not limited to two-liners and can be a lot more complex. Consider the following calculation :

def factorial(n):
    res = 1
    while n > 1:
        res = res * n
        n = n - 1
    return res

This function calculates the factorial of an integer. As you can see, we can use any code as before inside the function.

This example is also a good time to introduce another very important concept in programming : scope. The scope defines the longevity of a variable : when it is valid and when it is not. The principle is pretty simple : every variable declared inside a function is valid only inside the function. For instance, trying to execute this will end up with a runtime error :

factorial(10)
print(res)

Indeed, the res variable has been declared inside the function and has no existence outside of it ! If you need to reuse a variable defined inside a function, make sure you return it. Also note that you can return multiple values, in which case the result will be considered as a tuple :

def multiple_returns(a):
    return a, a*2, a*a

res = multiple_returns(10)
print(type(res))
print(res)

# It is also possible to unpack directly the tuple :
a, double_a, a_squared = multiple_returns(10)

Now that we have talked about scope, let's add a caution word on the parameters of a function. Parameters in a function are references, that means that in some cases modifying the value of a parameter in a function will modify it outside of the function. These cases solely depend on the type of parameters you provide. The types that will NOT be modified are : numbers, strings. All other types, especially lists and tuples WILL be modified outside if modified inside. Consider the following examples :

def modify1(a):
    a = 12
    return a

my_var = 7
b = modify1(my_var)
print(my_var) # 7 -> my_var has not been modified

def modify2(a):
    a[1] = 10 # We consider a as an indexed object such as a list !
    return a

my_var = [0, 1, 2, 3, 4]
b = modify2(my_var)
print(my_var) # [0, 10, 2, 3, 4] -> my_var has been modified

Remember what we said in Unit 2 about lists and references. If you really need to modify a list in a function, use the copy function to make a copy and modify the copy instead of modifying the original !

Modules

If you have a collection of functions then it can be useful to group them all together. That what's behind the concept of modules in Python. A module is very roughly a file containing collections of functions and variables that can be imported in a Python program. Python has, by default, a collection of modules that are very useful for developing programs. One module you will find unavoidable is the math module. Other very useful modules are os and sys (to operate on the system, navigate through files, etc.), itertools (to do more complex operations on lists). [Note : The modules mentioned here are listed for your general knowledge but it is not required that you know and/or use them].

Let's first see how to use modules. As said before, when we use modules, we import objects. Mostly functions and constants. There are various way to import data from a module. Consider the following commands :

# 1- Importing the whole package :
import math

# We can now use it by calling math.<object name>
print(math.pi)
print(math.sqrt(9))    

# 2- Importing the objects you need
from math import cos, sin

# We can use the objects we have imported directly :
print(cos(math.pi / 2.0))     
print(sin(math.pi * 1.5))

# 3- Importing everything from a module
from math import *

# Everything is usable without the reference to the module anymore :
print(gcd(10, 3))

All of these ways to import data from a package are useful but you should avoid as much as possible using the third way. Importing everything, when dealing with lots of functions from lots of different modules might have undesirable side-effects (replacing the function of a module by another one for instance).

Note that the first method can sometimes be a bit cumbersome. Imagine that you want to use a module called my_very_long_module that has three functions f1, f2 and f3. Such a code could be very heavy in no time :

import my_very_long_module

v1 = my_very_long_module.f1(10)
v2 = my_very_long_module.f2(my_very_long_module.f3(10), 7)

In such a case, Python allows us to alias module names with another name to help with the code :

import my_very_long_module as m

v1 = m.f1(10)
v2 = m.f2(m.f3(10), 7)

Now you might wonder how we create a module ? The answer could not be simpler : you just put functions and constants in a file and that's it. Let's work a simple example. Remember the add function we coded above ? Let's put it in a file called my_module.py. Let's add a sub function to it. So our file my_module.py should look like :

def add(a, b):
    return a+b

def sub(a, b):
    return a-b

In a different file in the SAME folder, now you can import these functions directly. The module name to import will be the name of the file without the .py : 

from my_module import add, sub
print(add(7, 10))
print(sub(10, 7))

When coding modules, you can add documentation to your functions by adding comments at the beginning of the function using triple quotes : '''. For instance :

def add(a, b):
    ''' This function adds two values and return the result '''
    return a+b

These comments, when placed here are called docstrings. You will find more information on what you can do with these in the official documentation. These docstring then appear when you use the help function. Hence help(add) will yield :

Help on function add in module my_module:

add(a, b)
    This function adds two values and return the result

The help command works for every public module so feel free to use it to find out information on the functions you are using. For instance, you can try to import the math module and type help(math) or help(math.cos).

As a final example, let's see a small program that will use Heron's formula to compute the area of a triangle :

from math import sqrt # We import the square root function
    
def tri_area(a, b, c):
    ''' This function uses Heron's formula to compute the area of a triangle
    of sides a, b, c '''

    s = (a+b+c) / 2.0
    return sqrt(s*(s-a)*(s-b)*(s-c))

a = int(input('Enter side a : '))
b = int(input('Enter side b : '))
c = int(input('Enter side c : '))

print('The area of the triangle is : ', tri_area(a, b, c))
